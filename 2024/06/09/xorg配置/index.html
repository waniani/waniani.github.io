INTRODUCTION

xorg.conf,xorg.conf.d 是 Xorg X server 的配置文件

Xorg支持几种提供或获取配置和运行时参数的机制：命令行选项，环境变量，xorg.conf和xorg.conf.d配置文件，自动检测和回退到默认值。当以多于一种方式提供相同信息时，使用最高优先级机制。机制列表从最高优先级到最低优先级排序。请注意，并非所有参数都可通过所有参数都可通过所有方法提供。可用的命令行选项和环境变量（以及一些默认值）在Xserver 和 Xorg 手册中有介绍。大多数配置文件参数以及默认值如下所述。驱动程序和模块特定的配置参数在相关的驱动程序或模块手册页中描述。

旧的XInput部分不再被识别。

ServerLayout 部分位于最高级别。它们将把一次会话中使用到的输入和输出设备绑定在一起。 输入设备在 InputDevice sections 中描述。输出设备通常由多个独立的组件（例如，图形板和监视器）组成。 这些多个组件在 Screen sections 中绑定在一起，这些是由ServerLayout 部分引用的。 每个 Screen sections 将图形板和显示器绑定在一起。图形板在 Device sections 进行了说明，监视器在 Monitor sections 进行了说明。

配置文件关键字不区分大小写，忽略“_”字符。 大多数字符串（包括选项名称）也不区分大小写，对空格和“_”字符不敏感。

每个配置文件条目通常占用文件中的一行。 它们由一个关键字组成，可能后跟一个或多个参数，参数的数量和类型取决于关键字。参数类型有：

  • Integer 十进制，十六进制或八进制的整数
  • Real 一个浮点数
  • String 用双引号括起来的字符串

注意：十六进制整数值必须以“0x”开头，八进制值以“0”开头。

可以使用称为Option的特殊关键字向服务器的各种组件提供自由形式的数据。Option关键字需要一个或两个字符串参数。 第一个是选项名称，可选的第二个参数是选项值。 一些常用的选项值类型包括：

  • Integer 十进制，十六进制或八进制的整数
  • Real 一个浮点数
  • String 一个字符序列
  • Boolean 布尔值（见下文）
  • Frequency 一个频率值（见下文）

请注意，所有Option值，而不仅仅是字符串，必须用引号括起来。

布尔选项可以可选地具有指定的值。当未指定值时，选项的值为TRUE。 以下布尔选项值被识别为TRUE：

**1, on, true, yes

并且以下布尔选项值被识别为FALSE：

0, off, false, no

如果选项名称前缀为“No”，则选项值被否定。

Hz, k, kHz, M, MHz

当省略单位名称时，将根据该值和该值的适当范围的期望确定正确的单位。建议在使用频率选项值时始终指定单位，以避免在确定值时出现任何错误。

SERFLAGS SECTION

除了本节（下面描述）特有的选项外，ServerFlags 部分用于指定一些全局Xorg服务器选项。 本节中的entries都是Options，但为了兼容性目的，仍然可以识别某些旧样式条目。 那些旧样式条目在这里没有记录，并且不鼓励使用它们。 ServerFlags section是可选的，可以在其中指定的条目。

本节中指定的Options（“DefaultServerLayout” Option除外）可能会被活动ServerLayouts section中指定的Options覆盖。 使用命令行等效项的选项将被覆盖。 本 section 认可的选项有：

Option “DefaultServerLayout“ “layout−id“

这指定在缺少-layout命令行选项时使用的默认ServerLayout部分。

Option “NoTrapSignals” “boolean“

这防止Xorg服务器捕获一系列意外的致命信号并干净地退出。否则，Xorg服务器将死机并且丢弃发生故障的核心。默认行为是Xorg服务器完全退出，但仍然放置一个核心文件。一般来说，除非您正在调试Xorg服务器问题并知道如何处理后果，否则不要使用此选项。

Option “UseSIGIO” “boolean”

这控制Xorg服务器是否请求通过SIGIO信号处理程序（在某些平台上也称为SIGPOLL）报告来自输入设备的事件，或仅通过标准select（3）循环进行报告。默认行为是特定于平台的。通常，您不想使用此选项，除非您正在调试Xorg服务器，或解决特定的错误，直到它被修复，并了解后果。

Option “DontVTSwitch” “boolean”

这不允许使用Ctrl + Alt + Fn序列（其中Fn表示编号的功能键之一）。该序列通常用于切换到具有此功能的操作系统上的另一个“虚拟终端”。启用此选项时，该键序列没有特殊含义，并传递给客户端。默认值：关闭。
Option “DontZap” “boolean”

这不允许使用Terminate_Server XKB操作（通常在Ctrl + Alt + Backspace，取决于XKB选项）。此操作通常用于终止Xorg服务器。启用此选项时，操作不起作用。默认值：关闭。

Option “DontZoom” “boolean”

这不允许使用Ctrl + Alt + Keypad-Plus和Ctrl + Alt + Keypad-Minus序列。这些序列允许您在视频模式之间切换。启用此选项时，这些键序列没有特殊含义，并传递到客户端。默认值：关闭。

Option “DisableVidModeExtension” “boolean”

这将禁用xvidtune客户端使用的VidMode扩展的可用于更改视频模式的部分。默认值：启用VidMode扩展。

Option “AllowNonLocalXvidtune” “boolean”

这允许xvidtune客户端（和使用VidMode扩展的其他客户端）从另一个主机连接。默认值：关闭。

Option “AllowMouseOpenFail” “boolean”

这告诉mousedrv（4）和vmmouse（4）驱动程序不报告故障，如果鼠标设备无法打开/初始化。它对evdev（4）或其他驱动程序没有影响。默认值：false。

Option “BlankTime” “time”

设置屏幕保护程序的空白阶段的不活动超时。时间以分钟为单位。这相当于Xorg服务器的-s标志，并且可以在运行时使用xset（1）更改该值。默认值：10分钟。

Option “StandbyTime” “time”

设置DPMS模式的待机阶段的不活动超时。时间以分钟为单位，并且该值可以在运行时使用xset（1）更改。默认值：10分钟。这仅适用于VESA DPMS兼容的显示器，并且可能不受所有视频驱动程序的支持。它仅对具有“DPMS”选项设置的屏幕启用（请参见下面的MONITOR部分）。

Option “SuspendTime” “time”

设置DPMS模式的暂停阶段的不活动超时。时间以分钟为单位，并且该值可以在运行时使用xset（1）更改。默认值：10分钟。这仅适用于VESA DPMS兼容的显示器，并且可能不受所有视频驱动程序的支持。它仅对具有“DPMS”选项设置的屏幕启用（请参见下面的MONITOR部分）。

Option “OffTime” “time”

设置DPMS模式的关闭阶段的不活动超时。时间以分钟为单位，并且该值可以在运行时使用xset（1）更改。默认值：10分钟。这仅适用于VESA DPMS兼容的显示器，并且可能不受所有视频驱动程序的支持。它仅对具有“DPMS”选项设置的屏幕启用（请参见下面的MONITOR部分）。

Option “Pixmap” “bpp”

这将设置pixmap格式以用于深度24. bpp的允许值为24和32.默认值：32，除非驱动程序约束不允许这种情况（这是罕见的）。注意：当此值设置为24时，会导致某些客户端行为不正常。

Option “NoPM” “boolean”

禁用与电源管理事件有关的功能。默认值：在支持它的平台上启用PM。

Option “Xinerama” “boolean”

启用或禁用XINERAMA扩展。默认为禁用。

Option “AIGLX” “boolean”

启用或禁用AIGLX。默认情况下启用AIGLX。

Option “DRI2” “boolean”

启用或禁用DRI2。默认情况下禁用DRI2。

Option “GlxVisuals” “string”

此选项控制GLX模块设置的GLX视图的数量。默认值是典型值，它将建立GLXFBConfigs的典型子集，由驱动程序提供为GLX视觉效果。其他选项是最小的，它将设置GLX规范允许的最小集合，所有这些都将为所有GLXFBConfig设置GLX视觉效果。

Option “UseDefaultFontPath” “boolean”

包括默认字体路径，即使在xorg.conf中指定了其他路径。 如果启用，则也包括其他字体路径。 默认情况下启用。

Option “IgnoreABI” “boolean”

允许加载为不同的，可能不兼容的X服务器版本构建的模块。 默认情况下禁用。

Option “AutoAddDevices” “boolean”

如果禁用此选项，则不会从HAL或udev后端添加任何设备。 默认情况下启用。

Option “AutoEnableDevices” “boolean”

如果禁用此选项，则将添加设备（并发送DevicePresenceNotify事件），但不启用，从而将策略留给客户端。 默认情况下启用。

Option “Log” “string”

此选项控制在每条消息后日志是否刷新和/或同步到磁盘。 可能的值为flush或sync。 默认情况下取消设置。

MODULE SECTION

Module section用于指定应加载哪些Xorg服务器模块。 当Xorg服务器以静态形式构建时，将忽略此部分。 通常在此部分加载的模块类型是Xorg服务器扩展模块。 大多数其他模块类型在通过其他机制需要时自动加载。 Module section是可选的，可以在其中指定的所有条目也是可选的。

INPUTDEVICE SECTION

配置文件可能有多个InputDevice sections。 最近的X服务器使用HAL或udev后端进行输入设备枚举和输入热插拔。 如果正在使用热插拔（即启用AutoAddDevices），通常不需要在xorg.conf中提供InputDevice section。 如果启用热插拔，则使用鼠标，kbd和vmmouse驱动程序的InputDevice section将被忽略。

如果禁用热插拔，通常至少有两个：一个用于核心（主）键盘，一个用于核心指针。 如果缺少这两个缺省配置，将使用缺少的缺省配置。 在没有明确指定的核心输入设备的情况下，使用标记为CorePointer（或CoreKeyboard）的第一个InputDevice。 如果没有匹配，则使用使用“鼠标”（或“kbd”）驱动程序的第一个InputDevice。 最后的备用是使用内置的默认配置。 当前，默认配置可能无法按预期在所有平台上工作。

Option “Floating” “boolean”

启用时，输入设备设置为浮动，并且不通过任何主设备报告事件或控制光标。 该设备仅适用于使用X输入扩展API的客户端。 默认情况下禁用此选项。 选项CorePointer，CoreKeyboard，AlwaysCore和SendCoreEvents是选项Floating的反向（即SendCoreEvents“on”等效于Floating“off”）。

此选项仅控制启动行为，设备可能在运行时重新连接或设置为浮动。

Option “TransformationMatrix” “a b c d e f g h i”

为绝对输入设备指定3x3变换矩阵。 输入设备将绑定到矩阵中给定的区域。 在大多数配置中，“a”和“e”指定设备所绑定的区域的宽度和高度，“c”和“f”指定区域的x和y偏移。 值范围是0到1，其中1表示所有根窗口的宽度或高度在一起，0.5表示一半的面积等。值表示3×3矩阵，第一，第二和第三组三个值表示第一 ，第二和第三行。 单位矩阵为“1 0 0 0 1 0 0 0 1”。

POINTER ACCELERATION

对于定点设备，以下选项控制指针相对于物理设备运动如何加速或减速。 大多数这些可以在运行时调整，有关详细信息，请参阅xinput（1）手册页。 这里只讨论最重要的加速度选项。

Option “AccelerationProfile” “integer”

选择配置文件。通俗地说，配置文件决定了加速的“感觉”。更正式地说，它定义了传递函数（当前设备速度和加速控制作为实际加速度的函数）的构建方式。这主要是个人偏好的问题。

  • 0 classic（大部分兼容）
    经典模式，通常与大多数设备兼容。

  • -1 none（仅应用恒定减速）
    不应用加速，仅有恒定的减速。

  • 1 device-dependent（设备依赖）
    根据设备的特性进行加速。

  • 2 polynomial（多项式函数）
    使用多项式函数进行加速。

  • 3 smooth linear（平滑线性）
    先是平滑过渡，然后线性加速。

  • 4 simple（慢速时正常，否则加速）
    速度慢时正常移动，速度快时加速。

  • 5 power（幂函数）
    使用幂函数进行加速。

  • 6 linear（速度越快，加速越多）
    线性加速，速度越快，加速度越大。

  • 7 limited（类似线性，但在阈值处达到最大）
    类似于线性加速，但在某个阈值达到最大加速。

Option “ConstantDeceleration” “real”

使指针比正常速度慢 deceleration 倍。对高分辨率设备最有用。

Option “AdaptiveDeceleration” “real”

允许指针在速度慢时实际减速。最多会慢 adaptive deceleration 倍。可以实现精确的指针定位，而不牺牲速度。

Option “AccelerationScheme” “string”

选择方案，即底层算法。

  • predictable 默认算法（行为更可预测）
  • lightweight 旧的加速代码（如 X 协议规范中所述）
  • none 无加速或减速

Option “AccelerationNumerator” “integer”
Option “AccelerationDenominator” “integer”

设置加速因子的分子和分母。加速因子是一个有理数，与阈值一起使用，可以调整配置文件以适应用户需求。简单和有限配置文件直接使用它（即加速因子），对于其他配置文件，加速因子越高，指针速度越快。通常，1 表示无加速，值高达 5 是合理的。

Option “AccelerationThreshold” “integer”

设置阈值，该阈值大致为使加速生效所需的速度（通常为每 10 毫秒的设备单位）。然而，具体效果因配置文件而异。

INPUTCLASS SECTION

配置文件可能包含多个 InputClass 部分。这些部分是可选的，用于为自动添加的输入设备类提供配置。一个输入设备可以匹配多个 InputClass 部分。每个类可以覆盖前一个类的设置，因此最好将最通用的匹配部分放在前面。

在所有 InputClass 部分中，Identifier 条目是必需的。所有其他条目都是可选的。

Identifier 条目指定此输入类的唯一名称。Driver 条目指定要用于此输入设备的驱动程序名称。在检查完所有类之后，当使用可加载服务器时，将启用来自第一个 Driver 条目的 “inputdriver” 模块。

当一个输入设备被自动添加时，其特征会与所有 InputClass 部分进行检查。每个部分可以包含可选条目以缩小类的匹配范围。如果没有出现任何可选条目，InputClass 部分是通用的，并将匹配任何输入设备。如果出现多个此类条目，则它们必须全部匹配才能应用配置。

InputClass 部分中有两种类型的匹配条目。第一种类型允许将各种标记与设备的属性进行匹配。可以通过使用 ‘|’ 字符分隔参数来构造一个条目以匹配来自不同设备的属性。可以提供相同类型的多个条目，以在同一属性上添加多个匹配条件。例如：

MatchProduct “matchproduct“

此条目用于检查设备的产品名称中是否包含子字符串 “matchproduct”。

MatchVendor “matchvendor“

此条目用于检查设备的厂商名称中是否包含子字符串 “matchvendor”。

MatchDevicePath “matchdevice“

此条目用于检查设备文件是否匹配 “matchdevice” 路径名模式。

MatchOS “matchos“

此条目用于检查操作系统是否匹配不区分大小写的 “matchos” 字符串。此条目仅在提供 uname(2) 系统调用的平台上支持。

MatchPnPID “matchpnp“

设备的即插即用 (PnP) ID 可以与 “matchpnp” shell 通配符模式进行检查。

MatchUSBID “matchusb“

设备的 USB ID 可以与 “matchusb” shell 通配符模式进行检查。ID 的格式是小写十六进制数字，以 ‘:’ 分隔。这与 lsusb(8) 程序的格式相同。

MatchDriver “matchdriver“

将区分大小写的字符串 “matchdriver” 与设备当前配置的驱动程序进行检查。使用此条目的部分的排序很重要，因为除非驱动程序已由配置后端或先前的 InputClass 部分设置，否则不会匹配。

MatchTag “matchtag“

此条目用于检查由配置后端分配的标签是否匹配 “matchtag” 模式。如果 “matchtag” 中给出的至少一个标签与后端分配的至少一个标签匹配，则找到匹配。

MatchLayout “matchlayout“

将区分大小写的字符串 “matchlayout” 与当前活动的 ServerLayout 部分进行检查。空字符串 “” 匹配隐式布局，如果未找到命名的 ServerLayout 部分，则出现隐式布局。

第二种条目用于匹配设备类型。这些条目接受类似于 Option 条目的布尔参数。
MatchIsKeyboard “bool“
MatchIsPointer “bool“
MatchIsJoystick “bool“
MatchIsTablet “bool“
MatchIsTouchpad “bool“
MatchIsTouchscreen “bool“

当输入设备匹配 InputClass 部分时，任何 Option 条目都会应用于设备。一个特定于 InputClass 的 Option 条目被识别。有关其余 Option 条目的描述，请参见上面的 InputDevice 部分。

Option “Ignore” “boolean“

此可选条目指定设备应完全忽略，不添加到服务器中。当设备由其他程序处理且不应生成 X 事件时，这可能很有用。

DEVICE SECTION

配置文件可能包含多个 Device 部分。必须至少有一个用于正在使用的视频卡。

Device 部分的格式如下：

在所有设备部分中，Identifier 和 Driver 条目是必需的。所有其他条目都是可选的。

Identifier 条目指定了该图形设备的唯一名称。Driver 条目指定了用于此图形设备的驱动程序名称。当使用可加载服务器时，驱动程序模块 “driver” 将为每个活动的设备部分加载。如果设备部分被活动的屏幕部分引用，则认为设备部分是活动的。

设备部分识别一些与驱动程序无关的条目和选项，在此进行描述。并非所有驱动程序都使用这些与驱动程序无关的条目，而且其中许多不需要指定它们，因为信息是自动检测的。有关此信息的更多详细信息，请参阅各个图形驱动程序手册页面，并了解设备特定选项的描述。请注意，这里列出的大多数选项（但不包括其他条目）可以在屏幕部分而不是设备部分中指定。

BusID “bus−id“

此处指定了显卡的总线位置。对于 PCI/AGP 卡，bus-id 字符串的形式为 PCI:bus:device:function（例如，“PCI:1:0:0” 可能适用于 AGP 卡）。在单头配置中使用主显卡时，此字段通常是可选的。在多头配置或在单头配置中使用次要显卡时，此条目是强制性的。其主要目的是在设备部分和其所代表的硬件之间建立明确的连接。此信息通常可以通过运行 pciaccess 工具 scanpci 来找到。

Screen number

此选项对于一个单一的 PCI 实体可以驱动多个显示器的卡是强制性的（即，多个 CRTCs 共享一个图形加速器和视频内存）。每个头部都需要一个设备部分，并且此参数确定每个设备部分应用于哪个头部。数字的合法值范围是 0 到每个实体的总头数减 1。大多数驱动程序要求主屏幕（0）存在。

Chipset “chipset“

这通常是可选的条目，指定了图形板上使用的芯片组。在大多数情况下，此条目不是必需的，因为驱动程序将探测硬件以确定芯片组类型。除非驱动程序特定文档建议您这样做，否则不要指定它。

Ramdac “ramdac−type“

此可选条目指定图形板上使用的 RAMDAC 类型。这仅由少数驱动程序使用，在大多数情况下，它不是必需的，因为驱动程序将尽可能探测硬件以确定 RAMDAC 类型。除非驱动程序特定文档建议您这样做，否则不要指定它。

DacSpeed speed
DacSpeed speed−8 speed−16 speed−24 speed−32

此可选条目指定 RAMDAC 速度评级（通常印在 RAMDAC 芯片上）。速度以 MHz 为单位。当给定一个值时，它适用于所有帧缓冲像素大小。当给定多个值时，它们分别适用于帧缓冲像素大小为 8、16、24 和 32。这不是许多驱动程序使用的选项，只有当 RAMDAC 的速度评级与驱动程序中的默认值不同时，或者当驱动程序无法自动检测到正确的默认值时，才需要指定它。除非驱动程序特定文档建议您这样做，否则不要指定它。

Clocks clock …

指定图形板上的像素时钟。时钟以 MHz 为单位，并可以指定为浮点数。该值被内部存储为最接近的 kHz。时钟的顺序很重要。它必须与在图形板上选择它们的顺序相匹配。可以指定多个 Clocks 行，并且每个 Clocks 行连接起来形成列表。大多数驱动程序不使用此条目，它仅对具有非可编程时钟的一些旧型号板卡是必需的。除非驱动程序特定文档明确建议您这样做，否则不要指定此条目。

ClockChip “clockchip−type“

此可选条目用于指定具有可编程时钟发生器的图形板上的时钟芯片类型。只有少数 Xorg 驱动程序支持可编程时钟芯片。有关详细信息，请参阅相应的驱动程序手册页面。

VideoRam mem

此可选条目指定安装在图形板上的视频内存量。此值以 kBytes 表示。在大多数情况下，这不是必需的，因为 Xorg 服务器会探测图形板以确定此数量。只有当驱动程序特定文档指示可能需要时，才需要指定。

BiosBase baseaddress

此可选条目指定 VGA 板的视频 BIOS 的基址。此地址通常会自动检测到，只有在驱动程序特定文档建议时才应指定。

MemBase baseaddress

此可选条目指定图形板线性帧缓冲区的内存基址。此条目不被许多驱动程序使用，只有在驱动程序特定文档建议时才应指定。

IOBase baseaddress

此可选条目指定 IO 基址。此条目不被许多驱动程序使用，只有在驱动程序特定文档建议时才应指定。

ChipID id

此可选条目指定表示芯片类型的数字 ID。对于 PCI 卡，通常是设备 ID。这可以用于覆盖自动检测，但只有在驱动程序特定文档建议时才应这样做。

ChipRev rev

此可选条目指定芯片修订号。这可以用于覆盖自动检测，但只有在驱动程序特定文档建议时才应这样做。

TextClockFreq freq

此可选条目指定用于常规文本模式的像素时钟频率。频率以 MHz 表示。这很少使用。

Option “ModeDebug” “boolean“

启用将有关模式设置的附加调试信息打印到服务器日志中。

Options

Option 标志可以在设备部分中指定。这些包括驱动程序特定选项和与驱动程序无关的选项。前者在驱动程序特定文档中进行了描述。一些后者在关于屏幕部分的部分下面描述，它们也可以包括在此处。

MONITOR SECTION

配置文件可能有多个 Monitor 部分。通常应该至少有一个，用于当前使用的显示器，但当没有指定一个时，将创建一个默认配置。

Monitor 部分的格式如下：

Monitor 部分中唯一必需的条目是 Identifier 条目。

Identifier 条目指定了此监视器的唯一名称。Monitor 部分可用于提供关于监视器规格、特定于监视器的选项以及要与监视器一起使用的视频模式的信息。

对于启用了 RandR 1.2 的驱动程序，监视器部分可以与视频卡的特定输出相关联。通过使用视频驱动程序定义的输出名称加上监视器部分的标识符，可以通过在 Device 部分中添加以下格式的选项将监视器部分与输出相关联：

Option “Monitor-outputname“ “monitorsection“

（例如，对于 VGA 输出，使用 Option “Monitor-VGA” “VGA monitor”）

如果没有将监视器部分具体关联到输出，则如果存在监视器部分，服务器将将其与输出相关联，以保持对以前的单头配置的兼容性。

指定视频模式是可选的，因为服务器将使用监视器提供的 DDC 或其他信息自动配置可用模式的列表。当在 Monitor 部分中显式指定模式（使用 Mode、ModeLine 或 UseModes 关键字）时，不会包括具有相同名称的内置模式。然而，当内置模式的名称不同但满足监视器要求时，仍然会隐式包含它们。

可以在 Monitor 部分中使用的条目如下所述。

VendorName “vendor“

此可选条目指定监视器的制造商。

ModelName “model“

此可选条目指定监视器的型号。

HorizSync horizsync−range

给出监视器支持的水平同步频率范围。horizsync-range 可以是逗号分隔的离散值或值范围。值范围是由连字符分隔的两个值。默认情况下，值以 kHz 为单位。如果在行尾添加了 MHz 或 Hz，则可以指定为 MHz 或 Hz。此处给出的数据由 Xorg 服务器用于确定视频模式是否在监视器规格内。此信息应该在监视器手册中可用。如果省略此条目，则使用默认范围 28−33kHz。

VertRefresh vertrefresh−range

给出监视器支持的垂直刷新频率范围。vertrefresh-range 可以是逗号分隔的离散值或值范围。值范围是由连字符分隔的两个值。默认情况下，值以 Hz 为单位。如果在行尾添加了 MHz 或 kHz，则可以指定为 MHz 或 kHz。此处给出的数据由 Xorg 服务器用于确定视频模式是否在监视器规格内。此信息应该在监视器手册中可用。如果省略此条目，则使用默认范围 43−72Hz。

DisplaySize width height

此可选条目给出监视器图片区域的宽度和高度，单位为毫米。如果给出，这将用于计算屏幕的水平和垂直间距（DPI）。

Gamma gamma−value
Gamma red−gamma green−gamma blue−gamma

这是一个可选条目，用于指定监视器的伽马校正。它可以指定为单个值或三个单独的 RGB 值。这些值应在 0.1 到 10.0 的范围内， 默认值为 1.0。并非所有驱动程序都能够使用此信息。

UseModes “modesection−id“

包含在名为 modesection-id 的 Modes 部分中列出的模式集合。这使得该部分中定义的所有模式可供此监视器使用。

Mode “name“

这是一个可选的多行条目，用于为监视器提供视频模式的定义。在大多数情况下，这不是必需的，因为内置的 VESA 标准模式集将足以满足需求。Mode 关键字表示多行视频模式描述的开始。模式描述以 EndMode 关键字终止。模式描述由以下条目组成：

  • DotClock clock

用于该模式的点（像素）时钟速率。

  • HTimings hdisp hsyncstart hsyncend htotal

指定该模式的水平定时。

  • VTimings vdisp vsyncstart vsyncend vtotal

指定该模式的垂直定时。

  • Flags “flag” …

指定一组可选的模式标志，每个标志是双引号中的单独字符串。”Interlace” 表示模式是交织的。”DoubleScan” 表示每个扫描线都是双倍的模式。”+HSync” 和 “−HSync” 可用于选择 HSync 信号的极性。”+VSync” 和 “−VSync” 可用于选择 VSync 信号的极性。”Composite” 可用于指定在支持此功能的硬件上的复合同步。此外，在某些硬件上，”+CSync” 和 “−CSync” 可用于选择复合同步极性。

  • HSkew hskew

指定显示使能信号向右边缘偏移的像素数。并非所有驱动程序都使用此信息。如果需要覆盖服务器提供的默认值（如果有），则可能需要此选项。“Roving” 水平线表示此值需要增加。如果一条扫描线上的最后几个像素出现在屏幕左侧，则应将此值减少。

  • VScan vscan

指定每个扫描线在屏幕上绘制的次数。并非所有驱动程序都使用此信息。小于 1 的值视为 1，这是默认值。通常，上述提到的 “DoubleScan” 标志会将此值加倍。

ModeLine “name“ mode−description

此条目是 Mode 条目的更紧凑版本，也可用于指定监视器的视频模式。这是一个用于指定视频模式的单行格式。在大多数情况下，这不是必需的，因为内置的 VESA 标准模式集将足以满足需求。

模式描述分为四个部分，前三个部分是必需的。第一个部分是点（像素）时钟。这是一个指定模式的像素时钟速率的单个数字，单位为 MHz。第二个部分是四个数字的列表，指定了水平定时。这些数字是 hdisp、hsyncstart、hsyncend 和 htotal 值。第三部分是四个数字的列表，指定了垂直定时。这些数字是 vdisp、vsyncstart、vsyncend 和 vtotal 值。最后一个部分是一组标志，指定模式的其他特性。Interlace 表示模式是交织的。DoubleScan 表示每个扫描线都是双倍的模式。+HSync 和 −HSync 可用于选择 HSync 信号的极性。+VSync 和 −VSync 可用于选择 VSync 信号的极性。Composite 可用于指定在支持此功能的硬件上的复合同步。此外，在某些硬件上，+CSync 和 −CSync 可用于选择复合同步极性。上述 Mode 条目描述中提到的 HSkew 和 VScan 选项也可以在此处使用。

Option “DPMS” “bool“

此选项控制服务器是否应为此屏幕启用 DPMS 扩展的电源管理。默认情况下，启用扩展。

Option “SyncOnGreen” “bool“

此选项控制视频卡是否应在绿色颜色引脚上驱动同步信号。并非所有卡都支持此选项，大多数监视器也不需要。默认情况下为关闭。

Option “Primary” “bool“

此可选条目指定监视器应被视为主要监视器。（仅限支持 RandR 1.2 的驱动程序）

Option “PreferredMode” “name“

此可选条目指定一个模式作为监视器的首选初始模式。（仅限支持 RandR 1.2 的驱动程序）

Option “Position” “x y“

此可选条目指定监视器在 X 屏幕内的位置。（仅限支持 RandR 1.2 的驱动程序）

Option “LeftOf” “output“

此可选条目指定监视器应位于给定名称的输出（非监视器）的左侧。（仅限支持 RandR 1.2 的驱动程序）

Option “RightOf” “output“

此可选条目指定监视器应位于给定名称的输出（非监视器）的右侧。（仅限支持 RandR 1.2 的驱动程序）

Option “Above” “output“

此可选条目指定监视器应位于给定名称的输出（非监视器）的上方。（仅限支持 RandR 1.2 的驱动程序）

Option “Below” “output“

此可选条目指定监视器应位于给定名称的输出（非监视器）的下方。（仅限支持 RandR 1.2 的驱动程序）

Option “Enable” “bool“

此可选条目指定是否应在启动时打开监视器。默认情况下，服务器将尝试启用所有连接的监视器。（仅限支持 RandR 1.2 的驱动程序）

Option “DefaultModes” “bool“

此可选条目指定服务器是否应将支持的默认模式添加到此监视器提供的模式列表中。默认情况下，服务器将添加默认模式；仅当您可以保证 EDID 将始终可用，或者如果您已添加了服务器可以使用的自定义模式行时，才应禁用此选项。（仅限支持 RandR 1.2 的驱动程序）

Option “MinClock” “frequency“

此可选条目指定监视器支持的最小点时钟，单位为 kHz。

Option “MaxClock” “frequency“

此可选条目指定监视器支持的最大点时钟，单位为 kHz。

Option “Ignore” “bool“

此可选条目指定完全忽略监视器，并且不通过 RandR 报告。如果硬件报告了不存在的输出，则此选项很有用。（仅限支持 RandR 1.2 的驱动程序）

Option “Rotate” “rotation“

此可选条目指定给定监视器的初始旋转。旋转的有效值为 “normal”、”left”、”right” 和 “inverted”。（仅限支持 RandR 1.2 的驱动程序）

MODES SECTION

配置文件可能有多个 Modes 部分，或者一个也没有。这些部分提供了一种独立于 Monitor 部分定义视频模式集合的方式。Monitor 部分可以通过使用 UseModes 关键字包含这些部分提供的定义。在大多数情况下，Modes 部分是不必要的，因为内置的 VESA 标准模式集将足以满足需求。

Modes 部分的格式如下：

Identifier 条目指定此模式描述集合的唯一名称。Modes 部分中允许的其他条目是在上面的 Monitor 部分中描述的 Mode 和 ModeLine 条目。

SCREEN SECTION

配置文件可能有多个 Screen 部分。必须至少有一个用于正在使用的“屏幕”。一个“屏幕”代表一个图形设备（Device 部分）和一个监视器（Monitor 部分）的绑定。如果一个 Screen 部分被一个活动的 ServerLayout 部分引用，或者通过命令行选项 -screen 引用，那么它被认为是“活动的”。如果两者都不存在，则配置文件中找到的第一个 Screen 部分被认为是活动的。

Screen 部分的格式如下：

Identifier 和 Device 条目是必需的。其他条目都是可选的。

Identifier 条目指定此屏幕的唯一名称。Screen 部分提供了针对整个屏幕的特定信息，包括屏幕特定的选项。在多头配置中，将有多个活动的 Screen 部分，每个头一个。此部分可用的条目有：
Device “device−id“

这个必需的条目指定要用于此屏幕的 Device 部分。这是将特定的图形卡与屏幕绑定的方法。device-id 必须与配置文件中 Device 部分的 Identifier 匹配。

Monitor “monitor−id“

指定要用于此屏幕的监视器描述。如果未指定监视器名称，则使用默认配置。目前，默认配置可能不会在所有平台上按预期工作。

VideoAdaptor “xv−id“

指定要与此屏幕一起使用的可选 Xv 视频适配器描述。

DefaultDepth depth

指定服务器应默认使用的颜色深度。-depth 命令行选项可用于覆盖此设置。如果都未指定，则默认深度是驱动程序特定的，但在大多数情况下是 8。

DefaultFbBpp bpp

指定默认使用的帧缓冲布局。-fbbpp 命令行选项可用于覆盖此设置。在大多数情况下，驱动程序将选择最佳的默认值。唯一一种情况是深度为 24，其中一些硬件支持打包的 24 位帧缓冲布局和稀疏的 32 位帧缓冲布局，这种情况下甚至有一个值的选择。

Options

在 Screen 部分可以指定各种选项标志。一些是特定于驱动程序的，并在驱动程序文档中描述。其他一些是与驱动程序无关的，并最终将在这里描述。

Option “Accel”

启用 XAA（X Acceleration Architecture），这是一种机制，使视频卡的 2D 硬件加速功能可供 Xorg 服务器使用。此选项默认为开启状态，但如果驱动程序存在错误，则可能需要将其关闭。有许多选项可用于禁用特定的加速操作，下面列出了这些选项。请注意，如果操作未加速（无论是由于硬件不支持还是驱动程序不支持），禁用操作将不会产生任何效果。

Option “InitPrimary” “boolean“

使用 Int10 模块来初始化主图形卡。通常，只有次要图形卡才会使用 Int10 模块进行软启动，因为主图形卡已经在启动时由 BIOS 初始化。默认值：false。

Option “NoInt10” “boolean“

禁用 Int10 模块，这是一个使用 int10 调用初始化图形卡的 BIOS 的模块。默认值：false。

Option “NoMTRR”

禁用 MTRR（Memory Type Range Register）支持，这是现代处理器的一个功能，可以将视频性能提高到 2.5 倍。一些硬件存在有缺陷的 MTRR 支持，而一些视频驱动程序在使用 MTRR 时已知会出现问题。

Option “XaaNoCPUToScreenColorExpandFill”

禁用从存储在系统内存中的源模式（使用内存映射孔）进行加速的矩形扩展填充。

Option “XaaNoColor8x8PatternFillRect”

禁用使用全彩模式对矩形区域进行加速填充。

Option “XaaNoColor8x8PatternFillTrap”

禁用使用全彩模式对梯形区域进行加速填充。

Option “XaaNoDashedBresenhamLine”

禁用加速虚线 Bresenham 线条绘制。

Option “XaaNoDashedTwoPointLine”

禁用在两个任意点之间绘制加速虚线。

Option “XaaNoImageWriteRect”

禁用从系统内存传输完整彩色矩形模式到视频内存的加速操作（使用内存映射孔）。

Option “XaaNoMono8x8PatternFillRect”

禁用使用单色模式对矩形区域进行加速填充。

Option “XaaNoMono8x8PatternFillTrap”

禁用使用单色模式对梯形区域进行加速填充。

Option “XaaNoOffscreenPixmaps”

禁用绘制存储在离屏视频内存中的像素图的加速操作。

Option “XaaNoPixmapCache”

禁用在离屏视频内存中对模式的缓存。

Option “XaaNoScanlineCPUToScreenColorExpandFill”

禁用从存储在系统内存中的源模式进行加速的矩形扩展填充（逐行进行）。

Option “XaaNoScanlineImageWriteRect”

禁用从系统内存传输完整彩色矩形模式到视频内存的加速操作（逐行进行）。

Option “XaaNoScreenToScreenColorExpandFill”

禁用从存储在离屏视频内存中的源模式进行加速的矩形扩展填充。

Option “XaaNoScreenToScreenCopy”

禁用从视频内存的一部分复制到另一部分的矩形区域的加速操作。

Option “XaaNoSolidBresenhamLine”

禁用加速的实线 Bresenham 线条绘制。

Option “XaaNoSolidFillRect”

禁用矩形区域的加速实色填充。

Option “XaaNoSolidFillTrap”

禁用 Bresenham 梯形区域的加速实色填充。

Option “XaaNoSolidHorVertLine”

禁用加速的实色水平和垂直线条绘制。

Option “XaaNoSolidTwoPointLine”

禁用两个任意点之间的加速实线绘制。

每个 Screen 部分可以选择包含一个或多个 Display 子部分。这些子部分提供深度/fbbpp 特定的配置信息，所选择的一个取决于用于屏幕的深度和/或 fbbpp。Display 子部分的格式在下面的部分中描述。

DISPLAY SUBSECTION

每个 Screen 部分可以有多个 Display 子部分。”活动” Display 子部分是第一个与正在使用的深度和/或 fbbpp 值匹配的子部分，或者在没有指定深度或 fbbpp 值的情况下，第一个没有匹配的子部分。Display 子部分是可选的。当没有与正在使用的深度和/或 fbbpp 值匹配的子部分时，所有可以在此指定的参数都会回退到其默认值。

Display 子部分的格式如下：

Depth depth

该条目指定 Display 子部分应使用的颜色深度。通常会指定此条目，但可以省略以创建一个匹配所有的 Display 子部分，或者希望仅与 FbBpp 参数匹配时。允许的深度值范围取决于驱动程序。大多数驱动程序支持 8、15、16 和 24。一些还支持 1 和/或 4，而一些可能支持其他值（如 30）。注意：深度表示像素中实际用于确定像素颜色的位数。32 不是有效的深度值。大多数使用每像素 32 位的硬件只使用其中的 24 位来存储颜色信息，这意味着颜色深度为 24，而不是 32。

FbBpp bpp

该条目指定此 Display 子部分应使用的帧缓冲格式。仅在提供深度为 24 的配置且允许在 24 bpp 打包帧缓冲格式和 32bpp 稀疏帧缓冲格式之间选择时才需要此条目。在大多数情况下，不应使用此条目。

Weight red−weight green−weight blue−weight

此可选条目指定在深度为 16 时用于屏幕的相对 RGB 加权。对于允许多种格式的驱动程序，也可以通过命令行使用 -weight 选项进行指定（参见 Xorg(1)）。

Virtual xdim ydim

此可选条目指定要使用的虚拟屏幕分辨率。对于大多数驱动程序，xdim 必须是 8 或 16 的倍数，并且在单色模式下必须是 32 的倍数。如果不是这种情况，给定的值将被舍入。对于超出指定虚拟大小的视频模式将被拒绝。如果不存在此条目，则虚拟屏幕分辨率将设置为容纳 Modes 条目中给出的所有有效视频模式。某些驱动程序/硬件组合不支持虚拟屏幕。有关详细信息，请参阅相应的驱动程序特定文档。

ViewPort x0 y0

此可选条目设置初始显示的左上角。仅当虚拟屏幕分辨率与初始视频模式的分辨率不同时才相关。如果未提供此条目，则初始显示将位于虚拟显示区域的中心。

Modes “mode−name“ …

此可选条目指定要使用的视频模式列表。每个指定的 mode−name 必须在双引号内。它们必须与适当 Monitor 部分中指定或引用的模式（包括隐式引用的内置 VESA 标准模式）相对应。服务器将从此列表中删除不满足各种要求的模式。此列表中的第一个有效模式将是启动时的默认显示模式。有效模式列表在内部转换为循环列表。可以使用 Ctrl+Alt+Keypad−Plus 切换到下一个模式，并使用 Ctrl+Alt+Keypad−Minus 切换到上一个模式。如果省略此条目，则将使用适当 Monitor 部分引用的有效模式。如果 Monitor 部分不包含模式，则将从内置的 VESA 标准模式中进行选择。

Visual “visual−name“

此可选条目设置默认的根视觉类型。这也可以从命令行指定（参见 Xserver(1) 手册页）。深度为 8 时可用的视觉类型为（默认为 PseudoColor）：

  • StaticGray
  • GrayScale
  • StaticColor
  • PseudoColor
  • TrueColor
  • DirectColor

深度为 15、16 和 24 时可用的视觉类型为（默认为 TrueColor）：

  • TrueColor
  • DirectColor

并非所有驱动程序都支持这些深度的 DirectColor。

深度为 4 时可用的视觉类型为（默认为 StaticColor）：

  • StaticGray
  • GrayScale
  • StaticColor
  • PseudoColor

深度为 1（单色）时可用的视觉类型为 StaticGray。

Black red green blue

此可选条目允许指定“黑色”。仅在深度为 1 时受支持。默认为黑色。

White red green blue

此可选条目允许指定“白色”。仅在深度为 1 时受支持。默认为白色。

Options

Display 子部分中可以指定选项标志。这些选项可能包括特定于驱动程序的选项和独立于驱动程序的选项。前者在特定于驱动程序的文档中有描述。一些后者在关于 Screen 部分的部分中描述，它们也可能在此处包括。

SERVERLAYOUT SECTION

配置文件可以包含多个 ServerLayout 部分。一个“服务器布局”代表一个或多个屏幕（Screen 部分）和一个或多个输入设备（InputDevice 部分）的绑定，形成一个完整的配置。在多屏配置中，它还指定了各个屏幕之间的相对布局。如果通过命令行选项“-layout”或通过 ServerFlags 部分中的 Option “DefaultServerLayout” 条目引用它，ServerLayout 部分将被视为“活动”。前者优先于后者。如果不使用这些选项，则在配置文件中找到的第一个 ServerLayout 部分被视为活动部分。如果没有 ServerLayout 部分存在，则选择单个活动屏幕和两个活动（核心）输入设备，如上述相关部分所述。

ServerLayout 部分的格式如下：

每个 ServerLayout 部分必须有一个 Identifier 条目和至少一个 Screen 条目。

Identifier 条目指定了此服务器布局的唯一名称。 ServerLayout 部分提供了关于整个会话的特定信息，包括会话特定的选项。 ServerFlags 选项（如上所述）可以在此处指定，并且此处给出的选项会覆盖 ServerFlags 部分中给出的选项。

可以在此部分中使用的条目在此处描述。

Screen screen−num “screen−id” position−information

对于每个在会话中使用的屏幕，必须提供其中之一。screen−id 字段是必需的，指定了要引用的 Screen 部分。screen−num 字段是可选的，可以用于在多头配置中指定屏幕编号。当省略此字段时，屏幕将按照它们在列表中出现的顺序进行编号。编号从 0 开始，并且必须连续。position−information 字段描述了多个屏幕的位置。有许多不同的方法可以提供此信息：

  • x y
  • Absolute x y

这两者都指定了左上角的坐标为 (x, y)。绝对关键字是可选的。一些旧版本的 XFree86（4.2 及更早版本）不识别绝对关键字，因此最安全的做法是不使用它，只需指定坐标即可。

  • RightOf “screen−id“
  • LeftOf “screen−id“
  • Above “screen−id“
  • Below “screen−id“
  • Relative “screen−id“** x y

这些将屏幕相对于另一个屏幕的位置。前四个将屏幕放置在另一个屏幕的右侧、左侧、上方或下方。在右侧或左侧定位时，顶部边缘对齐。在上方或下方定位时，左侧边缘对齐。Relative 形式指定了屏幕原点（左上角）相对于另一个屏幕原点的偏移量。

InputDevice “idev−id“ “option“ …

每个正在会话中使用的输入设备都应该有一个此条目。通常至少需要两个，一个用于核心指针设备，另一个用于键盘设备。如果其中一个缺失，则使用上面在 INPUTDEVICE 部分中描述的方法搜索适当的 InputDevice 条目。idev−id 字段是必需的，并指定要引用的 InputDevice 部分的名称。可以指定多个选项字段，每个选项都在双引号中。在此处允许的选项是可以在 InputDevice 部分中给出的任何选项。通常，只有会话特定的输入设备选项才会在此处使用。最常用的选项是：

“CorePointer”
“CoreKeyboard”
“SendCoreEvents”

前两个选项通常用于指示核心指针设备和核心键盘设备，而后者用于指示发送核心事件。

Options

除了以下内容之外，还可以在此处指定 ServerFlags 部分中允许的任何选项。当相同的选项同时出现在两个位置时，此处给出的值将覆盖 ServerFlags 部分中给出的值。

Option “IsolateDevice” “bus−id“

将设备重置限制为指定的总线 ID。有关 bus−id 参数格式，请参阅 BusID 选项（在上面的 DEVICE SECTION 中描述）。此选项将覆盖 SingleCard（如果指定）。目前，只有 PCI 设备可以以这种方式隔离。

Option “SingleCard” “boolean“

与 IsolateDevice 类似，但使用布局中第一个设备的总线 ID。

DRI SECTION

这个可选的部分用于为直接渲染基础设施提供一些信息。关于此部分格式的详细信息可以在 http://dri.freedesktop.org/ 线上找到。
